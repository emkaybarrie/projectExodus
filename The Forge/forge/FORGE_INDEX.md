# MyFi Forge Index

This directory contains the canonical institutional artifacts that govern how work is done.

## Canonical artifacts
- FORGE_KERNEL.md — constitution (process law)
- FORGE_STATE.md — current operating state (dual-track)
- FORGE_CAPSULE.md — portable summary for cross-agent use
- TASK_WORK_ORDER.md — mandatory intake format for all work
- FORGE_LESSONS.md — institutional memory (narrative + structure)
- FORGE_EVOLUTION.md — controlled self-improvement protocol
- FORGE_ROADMAP.md — capability matrix toward automation

## Product domain
The MyFi product is onboarded under `/myfi/`. Start with:
- /myfi/MYFI_ONBOARDING.md
- /myfi/PRODUCT_STATE.md
- /myfi/MYFI_CAPSULE.md

## Operating rule
If a task is requested informally, it must be converted into a Work Order (see TASK_WORK_ORDER.md) before execution.
